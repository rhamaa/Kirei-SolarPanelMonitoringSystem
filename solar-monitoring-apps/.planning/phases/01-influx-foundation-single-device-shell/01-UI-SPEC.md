## Interaction Contract (Phase 1 scope)

### Page + layout
- **Route**: `/dashboard` is the main shell for Phase 1 (redirect from `/` is allowed).
- **Max width**: 1024px content column, centered; full-bleed background.
- **Top bar (sticky)**: title + selectors; stays visible during scroll.

### Device selection (UI-03)
- **Control**: native `<select>` (or equivalent) + optional “Manual entry” text input is allowed, but Phase 1 MUST support selecting a `device_id`.
- **Default**: preselected from server-provided default (`DEFAULT_DEVICE_ID` from env via server payload).
- **Validation**: if user enters/chooses invalid/empty `device_id`, show inline error under selector and disable “Refresh KPIs”.

### Time range selection (UI-03)
- **Presets (minimum)**: `24h` (default) plus optional `1h`, `6h`, `7d` if easy.
- **Custom range**: not implemented in Phase 1; if rendered, it MUST be visibly disabled with helper text: “Custom range is coming in Phase 2.”

### KPI cards (UI-01)
- **Grid**: 2 columns on desktop, 1 column on small screens.
- **Each KPI card shows**:
  - Label (e.g., “PV Voltage”)
  - Value (large, display type)
  - Unit (label style)
  - Timestamp (“Last updated …”) in muted body text
- **Loading**: skeleton blocks for value + timestamp (not spinners).
- **Error**: inline red “Failed to load KPIs” message with a “Retry” link/button (secondary style).

### Env/config failures (OPS-01)
- **Missing env / invalid env** is a first-class UI state:
  - Render a full-width callout at top of dashboard (or dedicated error page) with:
    - Heading: “Influx isn’t configured”
    - Body: 1–2 sentence explanation
    - Bullet list of required vars (names only; never show actual values)
    - Action: “Open `.env.local` docs” (link to local README/section once Phase 3 exists; for Phase 1, omit link or point to a short inline snippet)
- Must never display token contents (even partially).

### Security boundary cues (OPS-02)
- The UI must not include any fields to input Influx connection details (URL/org/bucket/token). Only `device_id` + range are user-controllable.
